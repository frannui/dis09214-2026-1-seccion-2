## Pilares: descomposición, reconocimiento de patrones, la abstracción, algoritmos.
## Descomposición:
Dividir para conquistar, un problema complejo y grande, se rompe en partes pequeñas y
manejables. se hace un poco en procesos para llevarlo de manera más liviana, se
descompone en diseño A, B, C y códigos en las funciones propias, no solo se pone todo en
draw, si no se ponen los títulos para ordenar.
reconocimiento de patrones:
Si algo se repite o sigue una lógica constante se puede optimizar. ....
Abstracción:
Lo importante vs el detalle, filtrar la información que es innecesaria, solo definir las
características que definen el problema y crear una representación simbólica, se usa a
través de variables y mapa
## algoritmos:
receta de paso a paso, plan de acción que debe seguir el sistema, a traves de reglas
ordenadas. Se traduce en el diagrama de flujo y condicionales (if/else), mapa lógico que
conecta todas las partes anteriores.
